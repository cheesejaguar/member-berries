# Sample Project

This is a fictional example project showing what a repo-local `.pi/` memory setup can look like in practice.

## Project fiction

The sample repo is a small TypeScript API called **Acme Notes API**.

It provides:
- note creation and retrieval
- workspace-level note organization
- background embedding jobs for search
- a small admin UI for support/debugging

## Why this example exists

It demonstrates:
- what belongs in `PROJECT.md`
- what belongs in `current.md`
- how decisions differ from gotchas
- how commands and checkpoints should look

## What to copy from here

Use this example as a style guide, not a strict schema.

Copy the structure and tone:
- concise `PROJECT.md`
- action-oriented `current.md`
- explicit rationale in `decisions.md`
- operational scars in `gotchas.md`
- only verified commands in `commands.md`
- short factual snapshots in `checkpoints/`
